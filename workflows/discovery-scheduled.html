Stage 04 — Opportunity

Discovery — Scheduled

Platform GoHighLevel
Trigger sources 5 calendars (incl. Retell.AI)
Opportunity value $565

Purpose

This automation runs the moment a discovery call is booked, across five different calendar sources (including the Retell.AI voice booking flow). It first checks whether the contact is already a client or an existing SQL — if not, it creates or refreshes a $565 opportunity in the discovery pipeline, clears out any stale discovery/enrollment status tags, sets lifecycle to Opportunity, tags the contact as discovery-scheduled, and records which advisor the meeting was booked with. If the contact is already a client, it skips all of that and instead sends an internal notification so the team can reassign the conversation to the right customer success specialist instead of treating it as a new sales opportunity.

Simplified overview

Trigger: Appointment booked (5 calendar sources) │ ▼ IF lifecycle != Client AND tags do not contain client tags │ ├─ YES → Create $565 opportunity → Wait 1 min → Remove 19 stale tags │ → Lifecycle = Opportunity → Add tag "discovery - scheduled" │ → Update advisor assigned + appointment status │ → Find opportunity (latest) │ ├─ FOUND → Update opportunity + owner + advisor field │ └─ NOT FOUND → No action │ └─ NO → IF lifecycle == Client ├─ YES → Internal notification: reassign to CSS └─ NO → No action defined
