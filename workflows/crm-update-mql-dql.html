Purpose

This automation runs whenever a lead submits one of the two short credit forms. It reads the contact's self-reported credit tier and routes them down one of two paths: below 640 gets tagged and lifecycle-staged as Marketing Disqualified Lead, clearing any prior MQL status. 640 and above gets tagged and lifecycle-staged as Marketing Qualified Lead, with the submission date recorded. This is the single source of truth for whether a contact is allowed to continue toward booking a discovery call — the follow-up email sequence checks this exact MQL field before every send.

Simplified overview

Trigger: Form submitted (either short credit form) │ ▼ IF credit score is "Low: below 640" │ ├─ YES → Remove "MQL" tag → Add "MDQL" tag → Clear MQL field │ → Set MQL = No, set DQ date, set lifecycle = Marketing Disqualified Lead │ → End │ └─ NO → Remove "MDQL" tag → Add "MQL" tag → Set lifecycle = Marketing Qualified Lead → Set MQL = Yes, set MQL submit date, set MQL date → End
